# VowKy 1.2.0 更新内容

## AI 后处理（新功能）

- 转写完成后可让 AI 自动给原文加标题、摘要、段落分隔。AI 只输出「在哪里插入什么」的指令，由本地程序应用，绝不改动原文一个字。
- 同时并行生成三类元数据：标题、摘要、原音频路径、生成时间。
- 支持三种 AI 提供方：OpenAI 兼容 HTTP API、本机 Codex CLI、本机 Claude Code CLI。
- 在「设置 → AI 后处理」中可配置：开关、自动 / 手动触发、Provider、API Key / 二进制路径、超时、测试连接。

## 输出格式升级

- 转写文件后缀从 .txt 改为 .md（带 YAML frontmatter 的 Markdown）。旧的 .txt 文件保留在原目录不会被动到。
- 历史记录数据库新增 title / summary / audio_path / markdown_path / ai_provider 五个字段，搜索同时匹配正文、标题和摘要。

## 兼容与稳定性

- 老版本数据库自动迁移，无需手动操作。
- AI 后处理失败不影响转写本身保存；三个 AI 任务（标题 / 摘要 / 结构）单独失败也不阻塞其他任务。
- 超长文本（> 8000 字）的结构化任务自动按句子边界分块、客户端合并 operations。
