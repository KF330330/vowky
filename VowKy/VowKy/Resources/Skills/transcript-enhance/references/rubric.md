# transcript-enhance 终稿评估标准

Reviewer 按此评分。**通过 = Tier A 全 PASS && Tier B 总分 ≥ 95 / 100**。

## Tier A: 红线（任一 FAIL = 总 FAIL，不计 Tier B）

| 编号 | 红线 | 验证方法 |
|------|------|----------|
| A1 | byte-for-byte | `SHA256(input.txt) == SHA256(concat sentences.json[].text)` |
| A2 | 首段 H2 | 跳过 frontmatter + `## 目录` 段后，第一行非空 = `^## ` |
| A3 | frontmatter 字段集 | ⊆ {title, summary, audio_path, markdown_path, generated_at, duration_seconds} |
| A4 | 无 H1 | `grep -c '^# ' output.md` == 0 |
| A5 | .ai-log.txt 完整 | 含 outline / title / summary 三段 PROMPT+RESPONSE + SUMMARY 块 |
| A6 | ops.json 首条 | `operations[0].after_id == 0 && type == "h2"` |

## Tier B: 评分（满分 100）

### 维度 1 — 结构骨架（30 分）

#### 1.1 H2 数量（10 分）
内容 H2 数（**不含** TOC 的 `## 目录` heading）：
- ≤ 8: **10**
- 9: 7
- 10: 4
- 11-12: 1
- > 12: 0

#### 1.2 H2 主题切换准确率（10 分）
逐个 H2 判断：该 heading 引导的段落是否落在原文真实主题切换点？
- 错位率 = 错位 H2 数 / 总 H2 数
- 分数 = `round((1 − 错位率) × 10)`
- 错位例：把 H2 放在闲聊段 / 技术故障段 / 主题中段而非切换点

#### 1.3 H2 文字质量（10 分）
每条 H2 文字检查：
- ≤ 20 字（中英空格各算 1）
- 具体不泛泛（不含 "内容介绍"、"基础知识"、"其他模块" 等）
- 不以禁用后缀结尾：说明 / 建议 / 介绍 / 详解 / 概述 / 概要 / 情况 / 相关 / 知识 / 内容
- 不抄原文 ≥ 6 连续字（专有名词 Inside / Experience / Ptengine / Studio / UTM / AB / LP / PV / UV 等除外）

全合规：10；每违规 -2，扣至 0。

### 维度 2 — 子结构（20 分）

#### 2.1 H3 分布合理（5 分）
- 平均每 H2 0-4 个 H3 且分布均匀：5
- 极不均衡（有 H2 0 个 H3、有 H2 6+ 个 H3 并存）：3
- 异常（>50% H2 无 H3，或 H3 总数 > H2×5）：1

#### 2.2 H3 主题归属（10 分）
每 H3 必须在所属 H2 主题范畴内。
- 错位率 = 错位 H3 数 / 总 H3 数
- 分数 = `round((1 − 错位率) × 10)`

#### 2.3 H3 文字质量（5 分）
同 1.3 规则。全合规：5；每违规 -1。

### 维度 3 — 元数据（20 分）

#### 3.1 标题（6 分）
title 字段：
- ≤ 20 字 + 具体 + 概括全文：6
- 越界 / 泛泛 / 抄原文：每条 -2

#### 3.2 摘要长度（4 分）
- 200-500 字：4
- 150-199 或 501-600：2
- 严重越界（< 150 / > 600）/ 含 `\n` / 含禁用前缀：0

#### 3.3 摘要完整性（10 分）
必含元素：
(a) 主题
(b) 主要论点 / 内容范围
(c) 结论 / 受众 / 适用场景

- 三项齐全：10
- 缺一项：-3
- 缺两项：-6
- 缺三项：0

### 维度 4 — 排版可读性（20 分）

#### 4.1 段落呼吸（10 分）
检查每个"连续 sentence text 段"（两个 heading H2/H3 或 paragraph_break 或 H4 之间）：
- > 400 字：-1 each
- > 600 字：-2 each
- 最少扣到 0

H4 与 paragraph_break 等价地作为段落边界（H4 不进 TOC、不算 H2 数量、不进密度规则，只作内联可视分段）。

#### 4.2 标题密度（10 分）
相邻 heading 间字符数：100 ≤ x ≤ 800。
- 基础 10
- 每条违规 -2，扣至 0

### 维度 5 — 目录（10 分）

#### 5.1 TOC 完整性（4 分）
- 反映所有内容 H2 + H3：4
- 每个缺项 / 多余项：-1

#### 5.2 锚点链接正确（3 分）
- 每个锚点匹配 slug 算法（lower + 空格转 `-` + 标点移除 + CJK 保留）：3
- 每个错误：-1

#### 5.3 TOC 长度（3 分）
- ≤ 50 行：3
- 51-80：2
- > 80：1

## 通过条件

```
PASS = (Tier A 全 PASS) AND (Tier B 总分 ≥ 95)
```
